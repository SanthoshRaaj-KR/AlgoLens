# AlgoLens — Working Agreement

## Before Every Commit
1. Run the tests / exit criterion for the current phase
2. Explain to the user what was built and how it works
3. Only then commit

## Before Moving to the Next Phase
1. Confirm the current phase's exit criterion passes
2. Walk the user through what was built in plain English
3. Get acknowledgement before starting the next phase

## Why
The user wants to understand every piece as it lands — not just see code accumulate.
Stay synced. Never skip the explanation.
